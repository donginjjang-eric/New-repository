# auto_detail_page — 컨텍스트 노트

> 작업하면서 내린 결정과 그 이유를 계속 쌓는 문서.

## 2026-05-12 — 프로젝트 시작

### 결정 1: 스택은 Streamlit
- **이유**: 사용자가 이미 blogauto에서 Streamlit 사용 중. 학습 비용 0.
- HTML/CSS/JS 직접 작성 불필요 → 토스 스타일 미니멀 UI에 적합.
- 로컬에서 `streamlit run app.py` 한 줄로 서버 켜짐 (사용자 초보자 친화적).

### 결정 2: 이미지 생성 API는 gpt-image-1
- **이유**: 사용자가 "챗GPT 이미지 사용"이라고 명시. gpt-image-1이 GPT-4o 안에서 쓰는 그 모델.
- DALL-E 3 안 씀 (구형, 품질 떨어짐).
- 비용: 이미지 1장 약 0.04달러(high quality) → 5장 ≒ 0.2달러 ≒ 270원.

### 결정 3: 업로드 이미지는 GPT-4o Vision으로 분석
- **이유**: 사용자가 폼에 안 적어도 이미지만 보고 제품 정보를 추출할 수 있어야 함.
- 추출 결과는 폼에 자동 채워주고, 사용자가 수정 가능하게.

### 결정 4: 5단계 흐름은 "후킹→공감→특징→신뢰→구매유도" 공식
- **이유**: 사용자 선택. 가장 다년하고 검증된 상세페이지 흐름.
- `prompts.py`에 모듈화 → 나중에 사용자가 직접 텍스트만 수정 가능.

### 결정 5: 폴더 구조는 평평하게 (기존 참고자료와 같은 폴더)
- **이유**: 사용자 초보자. 폴더 깊어지면 어디에 뭐 있는지 헷갈림.
- 코드 파일은 `app.py`, `prompts.py`, `openai_client.py` 3개로 분리 (200줄 미만 유지).

### 결정 6: 한 파일에 다 넣지 않고 모듈 3개로 분리
- **이유**: 프롬프트는 사용자가 자주 손볼 가능성 높음 → 분리하면 수정 부담 적음.
- API 호출 로직은 따로 빼면 디버깅 쉬움.
- app.py는 UI만 담당 → 가독성 좋음.

## 알려진 제약사항
- gpt-image-2 응답 시간: 1장에 약 10~30초. 사용자가 기다리는 동안 로딩 표시 필수.
- OpenAI 무료 크레딧 5달러 → 약 125장 생성 가능 (테스트 충분).

## 2026-05-12 업데이트 — Stussy 티셔츠 테스트로 발견한 두 가지 문제 수정

### 문제 1: 입력 이미지가 모델에 안 넘어가고 있었음
- 첫 테스트(Stussy 검은 티셔츠 업로드)에서 결과가 "STREET" 글자가 박힌 완전히 다른 티셔츠로 나옴.
- 원인: `images.generate()` 만 사용 → 텍스트 프롬프트만 모델에 전달, 원본 이미지는 무시됨.
- 수정: `images.edit()` 로 변경 + 원본 이미지를 PNG로 변환해 함께 전달.
- 프롬프트 COMMON_STYLE에 "원본 상품의 디자인·로고·색상을 유지" 명시 추가.

### 문제 2: 모델 ID 업그레이드
- 사용자 요청에 따라 `gpt-image-1` → `gpt-image-2`로 변경.
- gpt-image-2는 2026-04-21 출시, 5월 초부터 API 사용 가능.
- 멀티모달 추론 강화 + 한국어 텍스트 처리 개선.

## 2026-05-12 UX 개선 — 초보자 친화 + 결과 보존

### 1~5번 버튼을 사이드바로 이동
- **이유**: 결과가 누적되어 스크롤이 길어져도 버튼이 항상 화면에 보임 ("따라다니게").
- 토스 좌측 네비 스타일.

### 사이드바 코드를 메인 코드 마지막에 배치 (중요)
- **이유**: Streamlit은 코드를 순차 실행함. 사이드바 버튼 클릭 → spinner 진입하면 그 시점에 아직 실행 안 된 아래쪽 메인 코드는 화면에 안 그려짐.
- 결과 영역 코드를 사이드바 코드보다 먼저 두면, 사이드바에서 spinner 도는 동안에도 이미 그려진 메인의 결과 카드가 그대로 유지됨.
- 사용자 피드백: "2번 만드는 동안 1번 자리에 이미지가 없어지면 안 돼" → 이 배치로 해결.

### 각 결과 카드에 "🔄 다시 만들기" 버튼
- **이유**: 결과가 마음에 안 들면 새로고침 없이 그 자리에서 재생성.
- 다운로드 버튼과 50:50 컬럼으로 나란히 배치.

### 사이드바 버튼에 ✅ 체크마크
- 이미 생성된 단계는 버튼 라벨에 ✅ 표시 → 진행 상황 한눈에.

## 2026-05-12 핵심 품질 개선 — 2단계 호출로 ChatGPT 품질 추격

### 문제 발견
- 같은 Stussy 이미지로 ChatGPT 웹에서 "상세페이지 1번 만들어줘" 한 줄 요청 시
  훨씬 디테일한 결과(도시 배경, 칩, CTA 띠 등) 나옴.
- 우리 앱은 정적 템플릿 프롬프트(`"후킹 메인 비주얼"`)만 전달 → 단순한 결과.

### 원인
- ChatGPT의 "Thought for 54s" = GPT가 thinking 단계에서 이미지 분석 +
  디테일한 이미지 프롬프트 자동 생성. 그 프롬프트를 gpt-image-2에 넘김.
- 우리 앱은 이 thinking 단계가 빠짐.

### 해결: 2단계 호출 구조 도입
1. **1단계**: GPT-4o (vision) + `DESIGNER_SYSTEM_PROMPT` →
   상품 이미지 + 단계 의도를 보고 detailed 한국어 이미지 프롬프트 생성
   (배경/레이아웃/카피 3종/컬러/폰트/상품 위치 7항목 포함)
2. **2단계**: 그 detailed 프롬프트 + 원본 이미지 → gpt-image-2 `images.edit()` 호출

### 비용·시간 영향
- 호출 1회당 GPT-4o vision 1번 추가 (≒ 0.005달러)
- 시간 20~40초 → 40~70초 (약 2배)
- 결과 디테일은 대폭 개선 예상

### 부가 UX
- 결과 카드에 "🧠 GPT가 만든 디자인 프롬프트 보기" expander 추가 →
  사용자가 어떤 프롬프트로 만들어졌는지 확인 가능 (학습/디버깅용).
- 사이드바 spinner에 "1/2: 디자인 프롬프트 → 2/2: 이미지 생성" 단계 안내.

## 2026-05-12 #3 — images.edit → images.generate 전환 + 프롬프트 대폭 강화

### 첫 시도 결과 (edit API)
- gpt-image-2의 `images.edit()`는 high fidelity 강제 → 원본 이미지를 거의
  변형 안 시킴. 결과는 원본 티셔츠 그대로 + 디자인 요소 거의 0.

### 두 번째 시도 (generate API)
- `images.generate()`로 전환. 원본 이미지는 1단계 GPT 분석에만 사용,
  2단계 생성에는 텍스트만 전달.
- 결과 디자인은 풍부해졌으나 카피·톤·단계 의도 위반 등 새로운 문제 발생:
  - "우아함의 정수" 같은 진부한 카피
  - 1번 후킹인데 "지금 구매하기" CTA 포함
  - 분홍 꽃 패턴 등 올드한 한국 홈쇼핑 감성
  - 한글 받침 깨짐("우아암의")

## 2026-05-12 #4 — 검수 에이전트 + Few-shot 학습 도입

### 사용자 요청
"점점 더 고퀄리티 상세페이지를 원함. 에이전트가 회의·검수·발행하고
학습되면 좋겠다."

### 결정 (사용자 답변)
- 범위: **가벼운 버전** — 검수 에이전트 1명 추가 (회의·발행 단계는 보류)
- 학습: **Few-shot 예시 누적** — ★ 받은 결과의 프롬프트를 JSON에 저장,
  다음 호출 시 디자이너 GPT에 참고로 제공

### 아키텍처
```
사용자가 N번 버튼 클릭
   ↓
[1단계: 디자인] GPT-4o + DESIGNER_SYSTEM_PROMPT + few-shot 예시
   ↓ detailed prompt (영문)
[2단계: 생성] gpt-image-2 (images.generate)
   ↓ 이미지 PNG
[3단계: 검수] GPT-4o vision + REVIEWER_SYSTEM_PROMPT
   ↓ 점수(0~100) + 부족한 항목 + 개선 제안
   ├─ 70점 이상 → 그대로 반환
   └─ 70점 미만 → 피드백 합쳐서 1회 재시도 → 반환
   ↓
[발행] 사용자가 👍 누르면 learning_pool.json에 저장
       → 다음 호출 시 few-shot 예시로 활용
```

### 비용·시간 영향
- 검수 호출 1회 추가 → 시간 +10초, 비용 +0.005달러
- 재시도 발생 시 시간·비용 2배
- 평균: 1장당 50~90초, 약 70~100원

### 결정 — Few-shot 풀 관리
- 위치: `learning_pool.json` (gitignore 제외 안 함 — 사용자 자산이라 깃에 올라가도 OK)
- 구조:
  ```json
  {"good_examples": [
    {"stage": 1, "prompt": "...", "product_type": "blouse",
     "timestamp": "2026-05-12T..."}, ...
  ]}
  ```
- 같은 stage 최근 3개만 로드 (토큰 절약)

## 2026-05-12 #5 — ChatGPT의 실제 프롬프트 패턴을 그대로 베끼기
- 사용자가 ChatGPT에 "방금 만든 이미지 프롬프트 알려줘"로 직접 받아온
  실제 영문 프롬프트 공유.
- 결정적 발견 5가지:
  1) 전체 지시는 **영어**로, 한국어 마케팅 카피만 큰따옴표 안에
  2) 명시적 출력 포맷: `Layout:`, `Style:`, `Include prominent Korean
     marketing text:` 라벨로 섹션 분리
  3) `"Image A"` reference 라벨로 첨부 이미지 명시
  4) Negative instruction 강제 (`"Do not make a collage of multiple pages;
     produce only page N."`)
  5) 카피를 1단계 GPT가 완전히 작성 (헤드라인·서브·배지 3개·하단 띠 모두)
- `DESIGNER_SYSTEM_PROMPT`를 전면 영문화 + ChatGPT 출력 포맷 그대로 강제.
- Korean copy는 반드시 double quotes 안에만 출력하도록 룰.

### 세 번째 개선 (이전) — ChatGPT 결과와 비교 후 대폭 강화
- ChatGPT 웹 결과(베이지 새틴 + 안개꽃 + 알약 칩 3개 + 트렌디 카피)를
  비교 분석. 우리 앱이 빠뜨린 5가지 항목 도출.
- `STAGES` 각 단계에 `must`(필수 구성) + `forbid`(금지 요소) 필드 추가.
  특히 1~4번에 CTA 금지를 명시.
- `DESIGNER_SYSTEM_PROMPT` 대폭 개편:
  - 디자인 톤 가이드 (모던 미니멀, 베이지/크림 톤, 분홍 꽃 금지)
  - 타이포그래피 7단 위계 (영문 헤더 → 디바이더 → 메인 → 서브 → 칩 →
    상품 → 하단 띠)
  - 카피 가이드 (진부 표현 블랙리스트, 글자수 명시)
  - 한글 깨짐 보완 (받침 명시, 짧게 유지)
- `build_prompt`도 STAGES의 must/forbid를 전달하도록 재작성.
